# Security Policy

## 支持范围

安全修复优先面向最新版本。项目尚处早期，请在升级前阅读变更说明。

## 本地数据边界

PixelCrew 会读取 Codex 本地状态、任务摘要和被允许的成果路径。`/api/status` 因而可能包含私有项目信息。服务默认绑定 `127.0.0.1`；不要直接暴露到公网，也不要把本地配置或快照提交到公开仓库。

AI 秘书是显式可选功能。虽然会过滤常见路径、任务 ID 和密钥，任何自动脱敏都不能覆盖所有机密格式。高敏感项目应只使用规则秘书。

## 报告漏洞

请不要在公开 issue 中附带真实任务数据、路径或凭据。优先使用 GitHub Security Advisories 的私密报告功能；仓库尚未启用该功能时，请联系仓库维护者并只发送最小复现。

报告中请包含受影响版本、攻击前提、最小复现和建议修复。维护者会尽快确认，但当前不承诺正式 SLA。
